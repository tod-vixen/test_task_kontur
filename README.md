
# Решение задания для собеседования на позицию AQA (см. файл task.txt)

## Резюме

Ниже представлено полное решение задачи: структурированный, хорошо прокомментированный код автотестов, инструкция по запуску, документация и пример отчёта. Код написан с использованием паттерна Page Object Model (POM), который является стандартом в индустрии и делает тесты максимально читаемыми, логичными и простыми для понимания.

## Инструкция по установке и запуску тестов

1. **Клонируйте репозиторий** или скопируйте файлы проекта в рабочую директорию.
   ```bash
   git clone git@github.com:tod-vixen/test_task_kontur.git
   cd test_task_kontur
   ```
2. **Создайте и активируйте виртуальное окружение** (опционально, но рекомендуется):
   ```bash
   python -m venv venv
   source venv/bin/activate   # Linux/Mac
   venv\Scripts\activate      # Windows
   ```
3. **Установите зависимости**:
   ```bash
   pip install -r requirements.txt
   ```
4. **Настройте учетные данные**:
   - Создайте в корне проекта файл `.env`.
   - Скопируйте в него содержимое `.env.example`.
   - Замените учетные данные на реальные данные зарегистрированного пользователя сайта https://100-dreley.ru/.
     Если у вас нет аккаунта, предварительно зарегистрируйтесь на сайте.
5. **Убедитесь, что установлен браузер Chrome** (драйвер будет загружен автоматически).
6. **Запустите тесты**:
   ```bash
   pytest -v
   ```

## Документация тестов

**Тест:** `test_login`

**Цель:** Проверить полный цикл работы механизма авторизации и выхода на сайте 100-dreley.ru.

**Предусловия:**
- Наличие зарегистрированного пользователя на сайте.
- В файле `.env` заданы корректные `LOGIN` и `PASSWORD`.
- Доступность сайта и стабильное интернет-соединение.

**Тестовые данные:**
- Некорректная пара: логин `test`, пароль `test`.
- Корректная пара: загружается из `.env`.

**Шаги и ожидаемые результаты:**

1. **Переход на главную страницу**
   *Ожидается:* Страница загружена, отображается ссылка «Вход».

2. **Клик по ссылке «Вход»**
   *Ожидается:* Открыта страница авторизации, присутствуют поля для ввода логина и пароля.

3. **Ввод некорректных данных и отправка формы**
   *Действие:* Заполняются поля `test`/`test`, нажимается кнопка «Войти».
   *Ожидается:* Появляется сообщение об ошибке с текстом, содержащим «Неверный логин или пароль»

4. **Ввод корректных данных и отправка формы**
   *Действие:* Поля очищаются, вводятся валидные логин и пароль, нажатие «Войти».
   *Ожидается:* Успешный вход – на странице появляется ссылка «Выход» (ссылка «Вход» исчезает).

5. **Клик по ссылке «Выход»**
   *Ожидается:* Пользователь выходит из системы, отображается главная страница со ссылкой «Вход» и отсутствует ссылка «Выход».

**Постусловия:** Браузер закрывается (фикстура `driver`).

**Особенности реализации:**
- Для поиска элементов используются стабильные атрибуты. При изменении вёрстки сайта потребуется актуализация локаторов.
- Все ключевые действия обёрнуты явными ожиданиями (`WebDriverWait`) для устойчивости к задержкам загрузки.
- Корректные учётные данные не хранятся в коде, а загружаются из `.env` через `python-dotenv`.

**Возможные риски:**
- Если сайт изменит названия полей (`USER_LOGIN`, `USER_PASSWORD`) или классы сообщений об ошибке, тест упадёт с понятным сообщением.
- При отсутствии файла `.env` или переменных `LOGIN`/`PASSWORD` тест будет пропущен (`pytest.skip`).
